### Stubs:
- Stubs são objetos ou métodos que substituem as implementações reais durante o teste. Eles são usados para simular o comportamento de partes específicas do sistema real. Em DDD, um stub poderia ser usado para substituir uma camada de infraestrutura, como uma camada de acesso a banco de dados ou um serviço externo, durante o teste de uma unidade de aplicação. Dessa forma, você pode isolar o código da aplicação e testá-lo independentemente do banco de dados real ou dos serviços externos, garantindo que o teste se concentre apenas na lógica da aplicação.
- Por exemplo, se você estiver testando uma classe de serviço que interage com um banco de dados, você pode usar um stub para simular as respostas do banco de dados, permitindo que você teste o comportamento da classe de serviço sem depender do estado real do banco de dados.
